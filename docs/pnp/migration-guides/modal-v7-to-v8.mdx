# Migration Guide from v7 to v8 for Web3Auth Modal SDK

## General

This migration guide provides steps for upgrading from version 7 (v7) to version 8 (v8) of the Web3Auth Modal SDK.

### `CustomChainConfig`

#### `blockExplorer` is now `blockExplorerUrl`
With v8, `CustomChainConfig` accepts `blockExplorerUrl` instead of `blockExplorer`.

#### A new `logo` parameter is now added to `CustomChainConfig`

#### A new `isTestnet` parameter is now added to `CustomChainConfig`

In `CustomChainConfig`, a new parameter `logo` has been introduced which can be used to pass
the logo for the chain. Along with that, `isTestnet` has been introduced which can be used to define
whether the network is testnet or not.

```tsx
const chainConfig = {
    chainId: "0x3",
    displayName: "Solana Testnet",
    chainNamespace: CHAIN_NAMESPACES.SOLANA,
    tickerName: "SOLANA",
    ticker: "SOL",
    decimals: 18,
    rpcTarget: "https://api.testnet.solana.com",
    // highlight-start
    blockExplorerUrl: "https://explorer.solana.com/?cluster=testnet",
    logo: "https://images.toruswallet.io/sol.svg",
    isTestnet: true
    // highlight-end
};
```

#### CustomChainConfig optional params
In v8, parameters apart from `chainId`, `rpcTarget`, and `chainNamespace` are now optional.
Please note, while using [Wallet Services](@web3auth/wallet-services-plugin) you have to also pass
the optional parameters.

### `Web3AuthOptions`

#### A new mandatory `privateKeyProvider` parameter is now added to `Web3AuthOptions`

With v8, it's mandatory to provide `privateKeyProvider` in Web3AuthOptions for your corresponding chain namespace. To know more in-depth about providers,
have a look at your [Providers reference](/sdk/helper-sdks/providers).

#### New language support for `defaultLanguage`

With v8, you can now use Turkish as a `defaultLanguage` option in `uiConfig`.

#### chainConfig is now optional

With v8, `chainConfig` is now optional parameter. The SDK fetches the `chainConfig` from `privateKeyProvider`.

```tsx
const chainConfig = {
  chainId: "0x1", // Please use 0x1 for Mainnet
  rpcTarget: "https://rpc.ankr.com/eth",
  displayName: "Ethereum Mainnet",
  blockExplorerUrl: "https://etherscan.io/",
  ticker: "ETH",
  tickerName: "Ethereum",
  logo: "https://images.toruswallet.io/eth.svg",
};

// highlight-start
const ethereumPrivateKeyProvider = EthereumPrivateKeyProvider({
  config: {chainConfig}
});
// highlight-end

const web3auth = new Web3Auth({
  clientId: "", // Get your Client ID from the Web3Auth Dashboard
  web3AuthNetwork: "sapphire_mainnet",
  uiConfig: {
    // highlight-next-line
    // Access the LANGUAGES from @toruslabs/openlogin-utils
    defaultLanguage: LANGUAGES.tr, // Supported languages en, de, ja, ko, zh, es, fr, pt, nl
  }
  // highlight-next-line
  privateKeyProvider: ethereumPrivateKeyProvider,
});
```

## Other changes

### WalletConnect v2 Adapter
With v8, `getWalletConnectV2Settings` now accepts a list of string, instead of list of number for `chainId`.

##### Usage

```tsx
import { WalletConnectModal } from "@walletconnect/modal";
import { getWalletConnectV2Settings, WalletConnectV2Adapter } from "@web3auth/wallet-connect-v2-adapter";

const defaultWcSettings = await getWalletConnectV2Settings("eip155", ["1"], "projectId");
const walletConnectModal = new WalletConnectModal({ projectId: "projectId" });
const walletConnectV2Adapter = new WalletConnectV2Adapter({
  adapterSettings: { qrcodeModal: walletConnectModal, ...defaultWcSettings.adapterSettings },
  loginSettings: { ...defaultWcSettings.loginSettings },
});

web3auth.configureAdapter(walletConnectV2Adapter);
```

### Adapters

Two new adapters, `@web3auth/default-evm-adapter` and `@web3auth/default-solana-adpater` have been introduced to help
developers configure default external adapters quickly for EIP155 and SOLANA namespace.

#### `@web3auth/default-evm-adapter`
Default EVM adapter allows you to create & configure external adapters for EIP155,
 like `@web3auth/torus-evm`, `@web3auth/metamask`, and `@web3auth/wallet-connect-v2` easily.

##### Installation

```bash npm2yarn
npm install --save @web3auth/default-evm-adapter
```

##### Usage

```tsx
const chainConfig = {
  chainId: "0x1", // Please use 0x1 for Mainnet
  rpcTarget: "https://rpc.ankr.com/eth",
  displayName: "Ethereum Mainnet",
  blockExplorerUrl: "https://etherscan.io/",
  ticker: "ETH",
  tickerName: "Ethereum",
  logo: "https://images.toruswallet.io/eth.svg",
};

const privateKeyProvider = new EthereumPrivateKeyProvider({ config: { chainConfig } });

const web3AuthOptions: Web3AuthOptions = {
  clientId,
  chainConfig: { ...chainConfig, chainNamespace: CHAIN_NAMESPACES.EIP155 },
  web3AuthNetwork: WEB3AUTH_NETWORK.SAPPHIRE_MAINNET,
  privateKeyProvider: privateKeyProvider,
};

// highlight-start
const adapters = await getDefaultExternalAdapters({ options: web3AuthOptions });

adapters.forEach((adapter) => {
  web3auth.configureAdapter(adapter);
});
// highlight-end
```

#### `@web3auth/default-solana-adapter`

Default Solana adapter allows you to create & configure external adapters for SOLANA namespace,
like `@web3auth/torus-solana`, and `@web3auth/phantom` easily.

##### Installation

```bash npm2yarn
npm install --save @web3auth/default-solana-adapter
```

##### Usage

```tsx
const chainConfig = {
    chainId: "0x3",
    displayName: "Solana Testnet",
    chainNamespace: CHAIN_NAMESPACES.SOLANA,
    tickerName: "SOLANA",
    ticker: "SOL",
    decimals: 18,
    rpcTarget: "https://api.testnet.solana.com",
    blockExplorerUrl: "https://explorer.solana.com/?cluster=testnet",
    logo: "https://images.toruswallet.io/sol.svg",
    isTestnet: true,
};

const privateKeyProvider = new SolanaPrivateKeyProvider({ config: { chainConfig } });

const web3AuthOptions: Web3AuthOptions = {
  clientId,
  chainConfig: { ...chainConfig, chainNamespace: CHAIN_NAMESPACES.SOLANA },
  web3AuthNetwork: WEB3AUTH_NETWORK.SAPPHIRE_MAINNET,
  privateKeyProvider: privateKeyProvider,
};

const adapters = await getDefaultExternalAdapters({ options: web3AuthOptions });

adapters.forEach((adapter) => {
  web3auth.configureAdapter(adapter);
});
```
