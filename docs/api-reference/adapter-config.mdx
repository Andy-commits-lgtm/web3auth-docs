Web3auth by default contains a pre configured list of login adapters. It also provides flexibility to  add/remove adapters or to use some existing adapter with custom configuration. Process of adding a new adapter or configuring the adapter relies on same function i.e. `web3auth.configureAdapter`, this function takes a single argument of adapter's instance being used. Some helper functions to generate instance of each adapter with custom configuration are available in `@web3auth/core` package.

This document act as a reference for configuring various adapters available in web3auth. Following is the list of available adapters,along with helper methods and configuration examples:-


 ## `Openlogin Adapter`:-
--------
  - `Adapter Name`:- `Openlogin`
  - `Description`:- Openlogin Adapter allows user to login with openlogin. You can read more about openlogin's authentication [here](https://docs.tor.us/open-login/get-started).
  - `Method`:- `getOpenloginAdapter`
  - `Example`:-
  ```ts
     import `Web3Auth` class from  `@web3auth/web3auth`;
     const web3auth = new Web3Auth(options);
     const openloginAdapter = await getOpenloginAdapter({
          adapterSettings: {
            network: "testnet",
            clientId: "localhost-id",
            uxMode: "popup"
          },chainConfig:{
            chainNamespace: this.web3auth.namespace,
            blockExplorer: "https://explorer.solana.com",
            chainId: "0x3",
            displayName: "Solana devnet",
            rpcTarget: "https://api.devnet.solana.com",
            ticker: "SOL",
            tickerName: "Solana Token",
          }
        })
// it will add/update  the openloginAdapter in to web3auth class
    web3auth.configureAdapter(openloginAdapter);
  ```
  - Interface:-
  ```ts
  interface OpenloginAdapterOptions {
    chainConfig?: CustomChainConfig;
    adapterSettings?: OpenLoginOptions;
    loginSettings?: LoginSettings;
  }
  ```
  - Params:-

    | Parameter | type |
    | -------- | -------- |
    | chainConfig     | [CustomChainConfig](#common_configuration)     |
    | adapterSettings     | [OpenLoginOptions](https://docs.tor.us/open-login/api-reference/initialization#openlogin)     |
    | loginSettings     | [LoginSettings](https://docs.tor.us/open-login/api-reference/usage#login)     |


## `Torus Evm Wallet Adapter`:-
--------
  - `Adapter Name`:- `torus-evm`
  - `Description`:- Torus Wallet adapter allows to connect with torus wallet for evm chains. You can read more about torus wallet [here](https://docs.tor.us/wallet/get-started).
  - `Method`:- `getTorusEvmAdapter`
  - `Example`:-
  ```ts
     import `Web3Auth` class from  `@web3auth/web3auth`;
     const web3auth = new Web3Auth(options);
     const adapter = await getTorusEvmAdapter({
         adapterSettings: {
          buttonPosition: "bottom-left"
        },
        loginSettings: {
          verifier: "google"
        },
        initParams: {
          buildEnv: "testing"
        },
        chainConfig:{
          chainNamespace: CHAIN_NAMESPACES.EIP155,
          chainId: "0x3",
          rpcTarget: "https://ropsten.infura.io/v3/776218ac4734478c90191dde8cae483c",
          displayName: "ropsten",
          blockExplorer: "https://ropsten.etherscan.io/",
          ticker: "ETH",
          tickerName: "Ethereum",
        }
    })

    // it will add/update  the torus-evm adapter in to web3auth class
    web3auth.configureAdapter(adapter);
  ```
  - Interface:-
  ```ts
  // refer to https://docs.tor.us/wallet/api-reference/class to know more about interface field types
  interface TorusWalletOptions {
    adapterSettings?: TorusCtorArgs;
    loginSettings?: LoginParams;
    initParams?: Omit<TorusParams, "network">;
    chainConfig?: CustomChainConfig;
}
  ```
  - Params:-

    | Parameter | type |
    | -------- | -------- |
    | chainConfig     | [CustomChainConfig](#common_configuration)          |
    | adapterSettings     | [TorusCtorArgs](https://docs.tor.us/wallet/api-reference/class)     |
    | loginSettings     | [LoginParams](https://docs.tor.us/wallet/api-reference/class)     |
    | initParams     | [Omit<TorusParams, "network">](https://docs.tor.us/wallet/api-reference/class)     |

## `Wallet Connect V1 adapter`:-
--------
  - `Adapter Name`:- `wallet-connect-v1`
  - `Description`:- Wallet connect v1 adapter allows you login with wallet connect v1. You can read more about wallet connect [here](https://docs.walletconnect.com/).
  - `Method`:- `getWalletConnectV1Adapter`
  - `Example`:-
  ```ts
     import `Web3Auth` class from  `@web3auth/web3auth`;
     const web3auth = new Web3Auth(options);
     const adapter = await getWalletConnectV1Adapter({
         adapterSettings: {
          infuraId:"YOUR_INFURA_ID"
        },
        chainConfig:{
          chainNamespace: CHAIN_NAMESPACES.EIP155,
          chainId: "0x3",
          rpcTarget: "https://ropsten.infura.io/v3/776218ac4734478c90191dde8cae483c",
          displayName: "ropsten",
          blockExplorer: "https://ropsten.etherscan.io/",
          ticker: "ETH",
          tickerName: "Ethereum",
        }
    })

    web3auth.configureAdapter(adapter);
  ```
  - Interface:-
  ```ts
  interface WalletConnectV1AdapterOptions {
      adapterSettings?: IWalletConnectProviderOptions;
      chainConfig?: CustomChainConfig;
    }
  ```
  - Params:-

    | Parameter | type |
    | -------- | -------- |
    | chainConfig     | [CustomChainConfig](#common_configuration)     |
    | adapterSettings     | [IWalletConnectProviderOptions](https://docs.walletconnect.com/quick-start/dapps/client#initiate-connection) |

### `Common Configuration Interfaces`


```ts

type CustomChainConfig = {
  chainNamespace: ChainNamespaceType;
  /**
   * The chain id of the chain
   */
  chainId: string;
  /**
   * RPC target Url for the chain
   */
  rpcTarget: string;
  /**
   * Display Name for the chain
   */
  displayName: string;
  /**
   * Url of the block explorer
   */
  blockExplorer: string;
  /**
   * Default currency ticker of the network (e.g: ETH)
   */
  ticker: string;
  /**
   * Name for currency ticker (e.g: `Ethereum`)
   */
  tickerName: string;
};


```
