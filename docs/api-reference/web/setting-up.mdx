## Choosing the right SDK

You can choose between [@web3auth/web3auth](https://npm.io/package/@web3auth/web3auth) and [@web3auth/core](https://npm.io/package/@web3auth/core)
packages.

#### [@web3auth/web3auth](https://npm.io/package/@web3auth/web3auth)

This package provides main class for using default web3auth modal. It is a child class of core package. So you can still call all the functions
available in the core api reference. The package includes all the following

#### [@web3auth/core](https://npm.io/package/@web3auth/core)

This package provides the core logic for handling adapters within web3auth. This package acts as a manager for all the adapters. You should use this
package to build your custom UI on top of web3auth.

#### [@web3auth/base](https://npm.io/package/@web3auth/base)

This package gives access to common types and interfaces for web3auth.

#### [@web3auth/ui](https://npm.io/package/@web3auth/ui)

Use this package if you wish to make use of the default Web3Auth modal UI instead of writing your own custom UI
![Web3Auth Modal](/images/web3auth-modal.png)

#### [@toruslabs/openlogin](https://npm.io/package/@toruslabs/openlogin)

OpenLogin is a plug & play authentication suite that combines the simplicity of passwordless authentication with the security of non-custodial public
key infrastructure (PKI).

Head over to the [docs](https://docs.tor.us/open-login/get-started) to get started.
