## Logging in a User

### `login({provider})`

Trigger the login flow that navigate the user to a browser model that allows them to login with the selected login provider.

| Parameter        | Type                      | Mandatory                           | Description                                                                                                                                                                                                     |
| ---------------- | ------------------------- | ----------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `provider`       | `enum` of `LoginProvider` | `Yes` for Web3Auth React Native SDK | Login provider, supported values are `GOOGLE`, `FACEBOOK`, `REDDIT`, `DISCORD`, `TWITCH`, `APPLE`, `LINE`, `GITHUB`, `KAKAO`, `LINKEDIN`, `TWITTER`, `WEIBO`, `WECHAT`, `EMAIL_PASSWORDLESS`, `WEBAUTHN`, `JWT` |
| `loginProvider ` | `enum of LoginProvider`   | `Yes` for OpenLogin Expo SDK        | Login provider, supported values are `GOOGLE`, `FACEBOOK`, `REDDIT`, `DISCORD`, `TWITCH`, `APPLE`, `LINE`, `GITHUB`, `KAKAO`, `LINKEDIN`, `TWITTER`, `WEIBO`, `WECHAT`, `EMAIL_PASSWORDLESS`, `WEBAUTHN`, `JWT` |
| `redirectUrl`    | `String`                  | `Yes` for OpenLogin Expo SDK        | The redirectUrl that the Web3Auth login flow will use to redirect back to the app. It is not required to be specified in the bare React Native SDK since it will be specified in the `init()` method.           |

### Sample Response

```json
}
```

### Example

#### Web3Auth React Native SDK

```typescript
const state = await Web3AuthReactNativeSdk.login({
  provider: LoginProvider.GOOGLE,
});
```

#### OpenLogin Expo SDK

```typescript
const scheme = "openloginexposdkexampleexpo";

const resolvedRedirectUrl =
  Constants.appOwnership == AppOwnership.Expo || Constants.appOwnership == AppOwnership.Guest
    ? new URL(Linking.createURL("openlogin", {}))
    : new URL(Linking.createURL("openlogin", { scheme: scheme }));
const state = await openlogin.login({
  loginProvider: LoginProvider.GOOGLE,
  redirectUrl: resolvedRedirectUrl,
});
```

## Logging out a user

---

### `logout()`

:::info

Trigger the logout flow on Android. This is a no-op on iOS since the iOS SDK does not have logout imeplemnted for technical reasons.

:::

This method has no input and returns a `Promise` that resolves if the operation is successful, and rejects if the operation failed.

### Example

#### Web3Auth React Native SDK

```typescript
await Web3AuthReactNativeSdk.logout();
```

#### OpenLogin Expo SDK

```typescript
await openlogin.logout();
```
