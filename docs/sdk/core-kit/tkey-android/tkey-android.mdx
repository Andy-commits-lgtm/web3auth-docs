The Web3Auth [tKey Android SDK](https://github.com/torusresearch/tkey-rust-android) is a client-side library you can use with your Android app to
authenticate users using Web3Auth. This authentication can be achieved using any social logins Web3Auth provides or a custom authentication flow. For
using Web3Auth's tKey natively in Android, Web3Auth provides a tKey Android SDK written in Java.

### Requirements

- Android API version `24` or newer.
- Basic knowledge of Java or Kotlin Development.

:::note source code

Source code of the SDK can be found **[on GitHub](https://github.com/torusresearch/tkey-rust-android)**.

:::

:::info Quick Start

You can have a look at the **[example](https://github.com/torusresearch/tkey-rust-android-example)** made on top of this SDK and try it out yourself.

:::

## Installation

## Add Web3Auth's tKey to Gradle

In your project-level `build.gradle` or `settings.gradle` file, add JitPack repository:

```groovy
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
        // highlight-next-line
        maven { url "https://jitpack.io" } // <-- Add this line
    }
}
```

Then, in your app-level `build.gradle` dependencies section, add the following:

```groovy
dependencies {
    // ...
    // highlight-next-line
    implementation 'com.web3auth.tkey:0.1.0'
}
```

:::info Latest-SDK

Check the **[latest version of Web3Auth's tKey Android SDK](https://github.com/torusresearch/tkey-rust-android/releases)** and update accordingly.

:::

### Permissions

Open your app's `AndroidManifest.xml` file and add the following permission:

```xml
// highlight-next-line
<uses-permission android:name="android.permission.INTERNET" />
```

> Note: The `<uses-permission>` element must be a direct child of the `<manifest>` root element
