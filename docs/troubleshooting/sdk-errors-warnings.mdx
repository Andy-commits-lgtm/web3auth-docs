## WalletInitializationError

| Code   | Message                                   |
| ------ | ----------------------------------------- |
| `5001` | Wallet is not found                       |
| `5002` | Wallet is not installed                   |
| `5003` | Wallet is not ready yet                   |
| `5004` | Wallet window is blocked                  |
| `5005` | Wallet window has been closed by the user |
| `5006` | Incompatible chain namespace provided     |
| `5007` | Adapter has already been included         |
| `5008` | Invalid provider Config                   |
| `5009` | Provider is not ready yet                 |
| `5010` | Failed to connect with rpc url            |
| `5011` | Invalid params passed in                  |
| `5013` | Invalid network provided                  |

## @web3auth/node-sdk

### Constructor Errors

- chainNamespace is required
- Please provide a valid clientId in constructor
- chainId is required for non-OTHER chainNamespace
- rpcTarget is required for non-OTHER chainNamespace

### Initialization Errors

- chainConfig is required for Solana in constructor
- chainConfig is required for EVM chain in constructor
- Invalid chainNamespace: ${this.currentChainNamespace} found while connecting to wallet

### Connect Errors

- User has already enabled mfa, please use the @web3auth/web3auth-web sdk for login with mfa

Some other errors could be found during it's implementations are:

- Error: node results do not match at first lookup ("code":-32602, "message": "Input error", "data": "Verifier not supported"
- Error: Error occurred while verifying paramsencoding/hex: invalid byte: U+0079 'y'
- Error: Duplicate token found ( returned when using a used id_token )
